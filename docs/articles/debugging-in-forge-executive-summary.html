What actually triggers a signal

Different root causes need different prevention strategies.
1
Spec-expectation gap
The spec and operator expectations diverged.
Prevention: Tighter ACs, assumption verification, Devil's Advocate gate.
EA-050: DA caught missing --fix flag before code was written.
2
Model knowledge gap
The agent didn't understand the tool or environment.
Prevention: Document as institutional pattern (CI entry) applied to future sessions.
EA-004-007: WSL / PowerShell / Jinja2 cross-platform cluster.
3
Implementation error
The model simply made a mistake — typo, hallucination, bad arithmetic.
Prevention: Mechanical validation: syntax checks, formula checks, linting.
EA-049: 11 specs with wrong priority scores in one batch.

What this means for debugging

FORGE doesn't add a debugging phase.
It recognizes debugging is already embedded — and focuses on
what traditional debugging leaves on the table:
the signals.
Persistent: the why outlives the commit.
Cumulative: isolated bugs become visible patterns.
Actionable: repeated pain becomes structural fixes.
Frictionless: no extra ceremony — just the work you're already doing.
