## Template variables

These placeholders are substituted during bootstrap:

| Variable | Description | Source |
|---|---|---|
| `{{PROJECT_NAME}}` | Project name | PRD interview Q1 / existing project |
| `{{PROJECT_DESCRIPTION}}` | 1–2 sentence description | PRD interview Q2 |
| `{{REPO_URL}}` | Git remote URL | PRD interview Q3 |
| `{{PRIMARY_STACK}}` | Language/framework | PRD interview Q4 |
| `{{DATE}}` | Bootstrap date | Auto (YYYY-MM-DD) |

## Brownfield rules

1. **Never overwrite** an existing file — skip and report.
2. **Never delete** anything in the target project.
3. If `.gitignore` exists but lacks `tmp/`, suggest appending it.
4. If `CLAUDE.md` exists, skip it and remind the user to review for process kit compatibility.
