# Expected cross-level drift — canonical repo-root files vs template/ mirrors
# Files listed here have known, intentional differences between canonical source
# (repo-root: .forge/commands/, .claude/agents/, docs/process-kit/) and the
# template/ mirror (template/.forge/commands/, template/.claude/agents/,
# template/docs/process-kit/).
#
# Format: relative/path/to/file.md | rationale
#   - Path is relative to repo root.
#   - Both canonical path (e.g. .forge/commands/close.md) and mirror path
#     (e.g. template/.forge/commands/close.md) will match entries in this file.
# Lines starting with # are comments. Blank lines are ignored.
#
# IMPORTANT (Spec 481): an entry here PERMITS drift; it does NOT ASSERT that
# drift currently exists. The sync uses three-state handling:
#   - currently-identical (canonical == mirror at HEAD): permitted drift has
#     not been applied, so additive edits ARE propagated to the mirror.
#   - currently-divergent (canonical != mirror at HEAD): genuine intentional
#     drift — the sync skips it and never clobbers the mirror.
#   - newly-divergent (identical at HEAD, canonical changed, mirror now stale):
#     forge-parity.sh --check emits an advisory WARN naming the file.
# So listing a file here does not by itself block additive propagation while
# the two copies still match. See docs/process-kit/sync-runbook.md.
#
# Spec 270 — Generalized Cross-Level Sync. See docs/process-kit/sync-runbook.md.
# ---
