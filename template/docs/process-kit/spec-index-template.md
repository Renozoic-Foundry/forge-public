# Specs Index

This directory contains versioned, reviewable specs used for spec-driven development.

## Conventions

- Naming: `NNN-short-title.md`
- Status values: `draft`, `in-progress`, `implemented`, `closed`, `deprecated`
- Specs are rebuild guides. **Errors and corrections**: update the spec body to reflect correct current state and add a dated revision entry documenting the change. **New scope or a different problem**: create a new spec.
- Each spec includes objective, scope, requirements, acceptance criteria, test plan, and compatibility/deprecation notes.
- Each active spec identifies ownership roles (author/reviewer/approver/implementation owner).
- Use `docs/specs/_template.md` for new specs.
- Record cross-spec transitions in `docs/specs/CHANGELOG.md`.
