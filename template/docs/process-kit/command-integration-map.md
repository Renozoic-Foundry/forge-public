## Integration Point Guidance

When creating a new command spec, include these ACs:
1. **Inbound**: At least one existing command recommends or chains to the new command
2. **Outbound**: The new command's choice block references the logical next commands
3. **Reference listing**: The new command appears in `docs/command-reference.md` (auto-generated) and `docs/QUICK-REFERENCE.md`
4. **Where it fits**: The command's help text includes a "Where it fits" section showing its relationship to adjacent commands

## Planning Stage Command Ladder

```
/brainstorm     "What should I work on?"     → Agent looks inward at project signals
/explore        "Is this idea viable?"       → Agent investigates, produces research artifact
/interview      "Help me think through X"    → Agent asks questions, user discovers
/consensus      "Get multiple perspectives"  → Agent runs multi-role review
/spec           "I know what I want"         → Agent writes the spec
```
