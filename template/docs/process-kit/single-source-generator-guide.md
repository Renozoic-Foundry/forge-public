# Single-source generator guide (NC-1a)

> Spec 480. Establishes one canonical source for FORGE's behavioral payload, with every other surface generated from it and a parity gate that fails on drift.

## Canonical direction — LOCKED

The **repo-root tree is the single canonical source**. Everything else is a generated downstream artifact.

| Surface | Role | Generated by |
|---------|------|--------------|
| `.forge/commands/`, `.claude/agents/`, `docs/process-kit/` (repo root) | **CANONICAL source** | — (edit here) |
| `.claude/commands/` (plugin payload) | downstream generated | `forge-sync-commands.sh` |
| `template/.forge/commands/`, `template/.claude/agents/`, `template/docs/process-kit/` (Copier surface) | downstream generated | `forge-sync-cross-level.sh` |

Both the plugin payload **and** the `template/` Copier surface are rendered artifacts of the same canonical source. This direction is locked (ADR-449; corrected and locked by /consensus 480 R1) — it matches the topology the existing scripts already implement (`forge-sync-commands.sh` hardcodes `CANONICAL_DIR=.forge/commands/`; the multi-agent generation depends on it). Do **not** edit a downstream surface directly and expect it to survive: regeneration overwrites it from canonical.

This kills the historical 4-copy mirror tax (a payload file existing as up to four hand-maintained copies) and makes a future Claude Code plugin (Spec 463 / NC-1b) a clean rendered artifact rather than a fifth hand-maintained copy.

## Entry point: `forge-parity.sh`

`.forge/bin/forge-parity.sh` is a thin wrap over the two existing sync scripts. It does **not** reimplement generation — it delegates.

```bash
# Regenerate all downstream surfaces from the canonical source:
bash .forge/bin/forge-parity.sh

# Verify no drift (CI / pre-close gate). Non-zero exit if EITHER surface drifted:
bash .forge/bin/forge-parity.sh --check
```

- **Generate (default)**: runs `forge-sync-commands.sh` then `forge-sync-cross-level.sh`, regenerating `.claude/commands/` and the `template/` mirrors from canonical.
- **`--check`**: runs both scripts' `--check` modes and exits non-zero if either reports drift, naming the drifted surface. Bounded runtime — it reuses the existing O(tracked-files) `--check` modes; **no Copier re-render**.

## Equality semantics (per-script common subset)

Parity is **not** whole-payload byte-exactness — that would conflict with the deliberate frontmatter preservation and `.jinja` skip the scripts already encode. Each script enforces its own equality definition over its own common subset:

- **`.claude/commands/` mirrors** (`forge-sync-commands.sh`): **body-equivalence** — frontmatter-aware (Spec 329). The mirror may carry agent-specific frontmatter; only the command body must match canonical.
- **Cross-level `cp` mirrors** (`forge-sync-cross-level.sh`): **byte-equivalence** over the mirrored subset (`.forge/commands/`, `.claude/agents/`, `docs/process-kit/`). Project-owned files (per `.forge/update-manifest.yaml`) are never mirrored; intentional drift is recorded in `.forge/state/expected-cross-level-drift.txt`.
- **`.jinja` Copier-var files**: **excluded** (Spec 281/390). These are intentional Copier-time substitution sites; their canonical and template forms differ by design.

## Enforcement

Two gates run the same `forge-parity.sh --check`:

1. **CI**: `.github/workflows/template-parity.yml` fails the build on any drift between canonical and a generated surface.
2. **`/close` Step 2d^2**: the single-source parity gate runs `forge-parity.sh --check` before a spec closes, offering a `regen` action that runs `forge-parity.sh` to restore parity.

## Out of scope (separate follow-up specs)

- **Merging** `forge-sync-commands.sh` and `forge-sync-cross-level.sh` into one script — deferred (their accreted edge-case handling makes a merge high-risk).
- The plugin manifest / `claude plugin install` — Spec 463 (NC-1b).
- Inverting the canonical direction to make `.claude/` the source — rejected (fights the existing topology).
