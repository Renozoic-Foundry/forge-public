The question operators keep asking

When you open FORGE, who does what isn't obvious — and the answer is deliberately configurable.
"FORGE has a spec author, a devil's advocate, a CISO, a CFO, a CTO...
am I supposed to use those? Or do I just think through those things myself?"
— Common operator question, first week with FORGE
Both answers are valid. Neither is a mistake.
The split between you and the agents isn't a fixed line. It's a dial, and FORGE exposes the dial explicitly. This deck shows how the dial works — and what each setting costs you.

The logic: when each role fires

FORGE's Review Router picks 2–3 roles per spec based on its characteristics — never all of them.
/spec
PLANNING
Spec Author drafts · Devil's Advocate always reviews
+ CISO if auth/data · + CXO if UX · + CFO/CTO if complex · + CCO if Lane B
/implement
EXECUTION
Implementer executes within scoped file list
Human owns scope enforcement; DA gate must PASS first
/close
VALIDATION
Validator verifies ACs independently in isolated worktree
+ any roles registered in forge.role_registry with contexts: [close]
/consensus
ON-DEMAND
Multi-role vote: approve · concern · reject
All registered roles dispatched in parallel, structured ballot returned
/evolve
IMPROVEMENT
Signal-based, not calendar-based — fires when thresholds hit
CEfO · CQO · DA · COO vet proposals before operator sees them
Role dispatch is selective, not exhaustive.
The Review Router matches roles to spec characteristics — you don't run all 16 every time.

What is always human, no matter the autonomy level

These categories cannot be delegated. The gate framework calls them out explicitly.
1
Physical / practical logic
"Will this actually work in the real world?" — always human-judgment-required.
gate-categories.md:40 — "cannot be delegated regardless of autonomy level"
2
Aesthetic, tone, and UX judgment
Voice, readability, feel. Agents can propose, but humans decide.
Also: ethical / legal judgment, novel-situation assessment.
3
Irreversible external actions
git push, /close, destructive git ops, external API side-effects.
Must never execute based on inference — requires an explicit operator message.
4
Every gate approval
No command auto-chains past a gate that hasn't received explicit human approval.
Live (L1/L2) or async via messaging (L3/L4) — either way, human-signed.

The autonomy dial: five settings

Same roles, same gates — you choose how much the agents are trusted to chain without pausing.
L0
MANUAL
Human performs all work; agents advise only
FORGE as a process framework — no AI runtime dependency
L1
HUMAN-GATED
Default. Human approves every gate transition
Agents produce, operator reviews at each checkpoint
L2
SUPERVISED
Agent auto-chains mechanical steps; human approves decision points
Faster throughput on routine specs; judgment gates still human
L3
TRUSTED
Full spec cycles run through without pausing; human reviews async
Requires NanoClaw or equivalent async-messaging integration
L4
FULL
Agent self-directed within budget ceilings; human intervenes on exception
Still gated on human-judgment categories above — always
The dial is per-project and per-spec-lane.
You can run L4 on docs and L1 on anything touching auth.

One valid model: the operator plays every non-AI role

No Devil's Advocate dispatch. No CTO review. You think those thoughts yourself.
What you gain
Zero dispatch tokens. Each advisory review costs 2k–5k tokens; skipping them is cheap.
Lower latency. No waiting for parallel sub-agents to return a verdict.
Tighter mental model. You stay in the weeds of the spec instead of delegating the critique.
Perfect for solo operators who already hold deep context across all domains.
What you trade away
Adversarial rigor. DA's 6-domain checklist is easy to forget when you're the author and the critic.
Structured evidence. Agent reviews leave a paper trail — role blocks, JSON verdicts, signals. Your mental review doesn't.
Fresh perspective. A sub-agent hasn't seen your spec draft — it can't rubber-stamp what you already decided.
Scale. When a team or additional operators show up, your in-head process doesn't transfer.
If you're solo and context-rich today, operator-does-everything is a reasonable default.
When stakes rise — auth changes, external APIs, compliance — start dispatching the specialists.

The bottom line

FORGE doesn't prescribe who does what.
It names every role, scopes every tool,
and hands you the dial.
You decide which roles are human, which are agent.
16 roles named — you can dispatch them, play them, or skip them.
5 autonomy levels — L0 manual all the way to L4 exception-only.
One inviolable rule — judgment, irreversible actions, and gate approvals stay human.
Everything else is a configuration choice, not a framework prescription.
